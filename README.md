# Scout YouTube Agent

Scout is a Mastra agent that turns rough YouTube ideas into researched, decision-ready briefs from inside Slack.

It delegates focused questions to a Perplexity Sonar research sub-agent, streams progress in Slack, remembers conversations, and creates approved ideas in a Notion database.

## What It Does

1. Accepts an unstructured video idea in Slack.
2. Splits the idea into focused research questions about competition, demand, freshness, and positioning.
3. Delegates each question to a Perplexity-powered Mastra sub-agent.
4. Produces a brief with a verdict, audience, angles, evidence, titles, hook, and open questions.
5. Iterates on the brief in the same Slack thread using persistent memory.
6. Shows an approval card before creating the idea in Notion.
7. Can list public Slack channels and summarize recent messages from channels it has joined.

## Architecture

```text
Slack
  -> Mastra channel webhook
  -> Scout (OpenAI GPT-5.6 Terra through OpenRouter)
       -> Researcher sub-agent (Perplexity Sonar)
       -> Slack channel history tools
       -> Notion creation tool (requires approval)
  -> PostgreSQL memory in production / LibSQL locally
```

The production server is the Hono-based Node.js server generated by `mastra build` and deployed to Railway.

## Requirements

- Node.js 22.13 or newer
- An OpenRouter API key
- A Slack workspace and Slack app
- A Notion workspace, connection, and database
- A Railway account for production deployment

## Local Setup

Clone the repository and install dependencies:

```bash
git clone git@github.com:Quintui/scout-youtube-agent.git
cd scout-youtube-agent
npm install
```

Create your local environment file:

```bash
cp .env.example .env
```

Set these variables in `.env`:

```env
OPENROUTER_API_KEY=sk-or-...
SLACK_SIGNING_SECRET=...
SLACK_BOT_TOKEN=xoxb-...
MASTRA_API_KEY=generate-a-random-secret
NOTION_API_KEY=ntn_...
NOTION_DATABASE_ID=...
```

Generate the Mastra API key with:

```bash
openssl rand -hex 32
```

`DATABASE_URL` is optional locally. Without it, Scout stores local memory in `./mastra.db` using LibSQL.

Start the development server:

```bash
npm run dev
```

Mastra runs on `http://localhost:4111` by default.

## Slack Setup

The fastest setup path is the included `slack-manifest.yaml`.

1. Open [api.slack.com/apps](https://api.slack.com/apps).
2. Select **Create New App** and then **From a manifest**.
3. Choose your workspace.
4. Replace `<YOUR-PUBLIC-URL>` in `slack-manifest.yaml` with your public server host.
5. Paste the manifest and create the app.
6. Install the app to your workspace.

For local development, expose Mastra through a tunnel:

```bash
npx cloudflared tunnel --url http://localhost:4111
```

The Slack webhook URL is:

```text
https://<YOUR-PUBLIC-URL>/api/agents/slack-agent/channels/slack/webhook
```

Use that URL for both **Event Subscriptions** and **Interactivity & Shortcuts**.

### Slack Credentials

| Variable | Slack location |
| --- | --- |
| `SLACK_SIGNING_SECRET` | **Basic Information** → **App Credentials** → **Signing Secret** |
| `SLACK_BOT_TOKEN` | **OAuth & Permissions** → **Bot User OAuth Token** |

The bot token begins with `xoxb-`.

### Slack Events

The app subscribes to:

- `app_mention`
- `message.channels`
- `message.im`

The bot can only read channel history after it has joined that channel:

```text
/invite @Scout
```

It cannot read private conversations that it does not participate in.

## Notion Setup

1. Open [Notion integrations](https://www.notion.so/profile/integrations).
2. Create a connection using the **Access token** authentication method.
3. Copy the token into `NOTION_API_KEY`.
4. Open the target Notion database.
5. Open the page menu, select **Connections**, and add the connection.
6. Copy the database ID into `NOTION_DATABASE_ID`.

The Notion tool expects these database properties:

| Property | Type | Values used by Scout |
| --- | --- | --- |
| `Name` | Title | Video idea title |
| `Status` | Select | `Needs Research` |
| `Format` | Select | `Long form` |
| `Bucket` | Select | `AI concepts`, `AI SDK specific`, `Mastra specific`, `Product development` |
| `Excitement` | Select | `1` through `5` |
| `Confidence` | Select | `1` through `5` |
| `Effort` | Select | `1` through `5` |

The title property name is discovered automatically. The remaining property names and select values must match the table above.

## Railway Deployment

Install and authenticate the Railway CLI:

```bash
curl -fsSL agents.railway.com | sh
railway login
```

Create or link a project, then add a PostgreSQL service. On the application service, configure:

```env
OPENROUTER_API_KEY=...
SLACK_SIGNING_SECRET=...
SLACK_BOT_TOKEN=...
MASTRA_API_KEY=...
NOTION_API_KEY=...
NOTION_DATABASE_ID=...
DATABASE_URL=${{Postgres.DATABASE_URL}}
```

Deploy the application:

```bash
railway up
railway domain
```

`railway.json` configures:

```text
Build: npm run build
Start: node .mastra/output/index.mjs
Health check: /health
```

After Railway generates a domain, update the two Slack request URLs:

```text
https://<YOUR-RAILWAY-DOMAIN>/api/agents/slack-agent/channels/slack/webhook
```

## Storage And Memory

Scout keeps the latest 30 messages from each conversation thread in memory.

| Environment | Storage |
| --- | --- |
| Local without `DATABASE_URL` | LibSQL at `file:./mastra.db` |
| Railway with a PostgreSQL `DATABASE_URL` | Mastra `PostgresStore` |

Mastra creates and manages its PostgreSQL tables automatically. Slack threads receive separate thread IDs, so their conversation histories remain isolated.

## Security

- Slack webhook requests are verified with `SLACK_SIGNING_SECRET`.
- Other Mastra API routes require `Authorization: Bearer <MASTRA_API_KEY>`.
- `/health` remains public for Railway health checks.
- Notion writes require an explicit Approve action in Slack.
- Secrets belong in `.env` locally and Railway Variables in production.
- `.env`, build output, and local database files are excluded from Git.

Example authenticated API request:

```bash
curl https://<YOUR-DOMAIN>/api/agents \
  -H "Authorization: Bearer $MASTRA_API_KEY"
```

## Try It

Send Scout this message in Slack:

```text
New video idea, rough brain dump: I want to build an AI agent that lives in
Slack, researches YouTube ideas, and saves approved briefs to Notion. I am
worried Mastra is too niche as a keyword, and I am not sure whether this
should be a full tutorial or a shorter showcase. Research the competition,
demand, recent changes, and the strongest positioning.
```

After Scout returns the brief, continue in the same thread:

```text
Make the angle more useful for developers building their first production agent.
```

Then save it:

```text
Save this idea to Notion.
```

Scout will display an approval card before writing the page.
